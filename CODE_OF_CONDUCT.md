# Code of Conduct

## Purpose

This project aims to keep collaboration respectful, direct, and productive for contributors, users, and maintainers.

Participation in this repository, its issues, pull requests, discussions, and related community spaces is expected to follow this Code of Conduct.

## Expected Behavior

Contributors are expected to:

- communicate respectfully, even when disagreeing
- focus criticism on ideas, code, and behavior, not people
- assume good intent first, then ask for clarification when needed
- provide actionable technical feedback
- respect maintainer decisions about scope, quality, and project direction
- help keep security, privacy, and legal concerns handled responsibly

## Unacceptable Behavior

The following are not acceptable:

- harassment, intimidation, or personal attacks
- hate speech or discriminatory language
- abusive, insulting, or deliberately inflammatory comments
- doxxing or sharing private information without permission
- spam, trolling, or repeated bad-faith arguments
- knowingly submitting malicious code, exploit details in public, or unsafe instructions that put users at risk
- any behavior that makes participation unnecessarily hostile or unsafe

## Project Standards

This repository is a technical project. Strong disagreement is acceptable; disrespect is not.

Maintainers may reject contributions that are out of scope, unsafe, poorly supported, or unnecessarily disruptive. That is a project decision, not a personal judgment.

## Reporting

If you experience or witness serious misconduct:

- for security-sensitive issues, follow [SECURITY.md](SECURITY.md)
- for conduct issues, contact the maintainer privately through the primary repository contact path

When reporting, include enough detail for the maintainer to understand what happened.

## Enforcement

Maintainers may take any action they consider appropriate, including:

- requesting changes in tone or behavior
- removing comments, issues, or pull requests
- locking conversations
- rejecting contributions
- temporarily or permanently restricting participation

Enforcement decisions will be based on protecting the project and the people participating in it.

## Scope

This Code of Conduct applies to:

- GitHub issues
- pull requests
- discussions
- code review conversations
- other project-managed communication spaces

## Final Note

The goal is a repository where people can collaborate efficiently, debate technical decisions honestly, and still treat each other with basic respect.
