# Support

## Scope

This repository provides the PlayFab-Catalog-Service-Bedrock API and related tooling. Support is best-effort and focused on issues that can be reproduced, explained clearly, and are within the intended scope of the project.

## Before Requesting Help

Please check these first:

- [readme.md](readme.md) for setup, configuration, API behavior, and operational details
- [CONTRIBUTING.md](CONTRIBUTING.md) for development and contribution expectations
- [SECURITY.md](SECURITY.md) if your concern may be security-related
- existing GitHub issues and discussions to avoid duplicates

## When To Open What

Use the correct channel for the type of request:

- **Bug reports:** open a GitHub issue with clear reproduction steps
- **Feature requests:** open a GitHub issue describing the use case and expected behavior
- **Questions / support requests:** open a GitHub discussion or issue, depending on what is enabled for the repository
- **Security issues:** do **not** post publicly; follow [SECURITY.md](SECURITY.md)

## What To Include In A Support Request

The more concrete your report is, the easier it is to help.

Include where relevant:

- what you are trying to do
- which endpoint, feature, or workflow is affected
- exact error message or unexpected behavior
- steps to reproduce the problem
- request example, payload, query, or relevant headers
- response status code and body
- logs or stack traces with secrets removed
- Node.js version, npm version, OS, and deployment context
- relevant environment variables or feature flags, without exposing sensitive values

## Supported Help

Reasonable support typically includes:

- clarifying documented behavior
- identifying whether something is a bug, misuse, or unsupported scenario
- guidance for local setup problems that can be reproduced
- reviewing whether a requested feature fits the project scope

## Support Limits

Support may be limited or unavailable for:

- heavily modified forks
- private infrastructure issues outside this repository
- incomplete reports without enough detail to reproduce
- requests that require access to private data, secrets, or paid third-party environments
- urgent production incidents that require guaranteed response times

## Response Expectations

This is not a commercial support channel and no response-time guarantee is provided.

Issues and questions are handled as time permits. Well-structured reports with reproducible details are much more likely to get a useful response.

## Security And Sensitive Data

- Never post secrets, tokens, session data, credentials, or private keys in issues, discussions, or pull requests.
- Redact sensitive data from logs, screenshots, and request examples.
- If the issue could affect security, report it privately using [SECURITY.md](SECURITY.md).

## Conduct

Please keep support interactions respectful and technical. By participating, you agree to follow [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).
